# قائمة فحص الحماية قبل الإصدار

## الأسرار والبيئة

- [ ] لا يوجد `service_role` أو كلمة مرور قاعدة البيانات في المستودع أو APK.
- [ ] متغيرا الويب الوحيدان هما رابط Supabase والمفتاح العام.
- [ ] عنوان الإنتاج يعمل عبر HTTPS وتظهر رؤوس CSP وHSTS وDENY.
- [ ] مفاتيح توقيع Android محفوظة في GitHub Secrets وليست ملفات المشروع.
- [ ] حُدد `ANDROID_APPLICATION_ID` دائم خارج `com.example` قبل أول إصدار عام.
- [ ] أضيفت أسرار keystore الأربعة ورابط Supabase والمفتاح العام إلى GitHub Actions.

## Supabase والبيانات

- [ ] أخذ نسخة احتياطية قبل migrations.
- [ ] تطبيق migrations بالترتيب على مشروع مرحلي أولًا.
- [ ] اختبار معلمين من حلقتين والتأكد من منع القراءة والكتابة المتبادلة.
- [ ] اختبار التعديل والحذف والمزامنة من جهازين.
- [ ] استعادة نسخة احتياطية كاملة والتحقق من عدد الطلاب والسجلات.
- [ ] تطبيق migration P6.2 وظهور bucket الخاص `halaqah-backups` بقيمة `public = false`.
- [ ] حساب ثانٍ لا يستطيع عرض أو تنزيل نسخة الحساب الأول أو سجل حلقة أخرى.
- [ ] تجربة عبارة حماية خاطئة وملفًا عُدّل يدويًا والتأكد من رفض الاستعادة دون تغيير البيانات.
- [ ] استعادة النسخة المشفرة على جهاز ثانٍ ثم مقارنة آخر حفظ ومراجعة وخطة وتقارير طالب معروف.
- [ ] حفظ عبارة الاستعادة خارج الجهاز وعدم إدراجها في لقطات الشاشة أو التذاكر أو الرسائل.

## بوابات الجودة

- [ ] نجاح GitHub Actions: مهمة الويب ومهمة Flutter.
- [ ] نجاح `npm run quality:ci` دون تجاوز الحد.
- [ ] عدم وجود ثغرات npm عالية أو حرجة.
- [ ] نجاح `flutter analyze` و`flutter test` وبناء APK موقع.
- [ ] نجاح `apksigner verify` ومطابقة SHA-256 للملف المنزل مع ملف سير البناء.
- [ ] الاحتفاظ برموز فك التتبع `build/symbols` الخاصة بكل إصدار إنتاج مموه.
- [ ] فحص APK على جهاز حقيقي بإصدار Android قديم وحديث.
- [ ] التأكد أن الجهاز التجريبي Android 6.0 أو أحدث وأن Android Auto Backup العام معطل.

## اختبارات القبول

- [ ] الحضور والحفظ والمراجعة والخطط والاختبارات تعمل بحساب معلم حقيقي.
- [ ] التقارير A4 وA5 والكاشير مجربة على طابعة فعلية.
- [ ] المشاركة إلى WhatsApp والصور تعمل دون تسريب بيانات طالب آخر.
- [ ] الوضع الداكن والاستجابة والقائمة الجانبية مجربة على الهاتف والويب.
- [ ] نجاح «فحص اتصال Supabase» من Wi-Fi ومن بيانات الهاتف قبل اختبار المزامنة.
- [ ] مراجعة تقرير مركز التشخيص والتأكد أنه لا يحتوي أسماء أو هواتف أو ملاحظات أو مسارات جهاز.
- [ ] ظهور حوادث التشغيل كبصمة مختصرة فقط دون نص الاستثناء أو stack trace.
- [ ] سياسة الخصوصية والاحتفاظ والحذف منشورة ومقبولة قبل جمع بيانات حقيقية.
