# تسليم المرحلة P6.1 — الحماية وبوابات جودة الإصدار

التاريخ: 2026-07-13

## النتيجة

أصبح للمشروع خط أساس إنتاجي يمنع دمج كود يكسر البناء أو يعيد ثغرة عالية
الخطورة أو يزيد دين lint. كما أضيفت حماية HTTP وتعافٍ عربي من أخطاء الويب،
وحُدثت الاعتمادات المتأثرة بالتنبيهات الأمنية.

## حماية الويب

- CSP تقصر المصادر على التطبيق وSupabase، وتمنع `object` و`frame-ancestors`.
- `X-Frame-Options: DENY` و`nosniff` وسياسة إحالة وصلاحيات متصفح مقيدة.
- HSTS في بناء الإنتاج فقط؛ يجب نشر الموقع عبر HTTPS.
- الكاميرا مسموحة للمصدر نفسه كي تعمل شاشة QR، والميكروفون والموقع والدفع
  وUSB محظورة.
- لا ينشأ عميل Supabase دون HTTPS ومفتاح عام، مع السماح بـlocalhost للتطوير.
- لا يوجد مفتاح خدمة سري في كود المتصفح.

## التعافي وتجربة الخطأ

- `error.tsx` لخطأ المسار مع إعادة محاولة ورمز تتبع غير حساس.
- `global-error.tsx` لخطأ الجذر مع وثيقة HTML عربية مستقلة.
- `not-found.tsx` لصفحة 404 عربية تعيد المستخدم إلى الرئيسية.

## الاعتمادات

- Next.js: `16.2.4` ← `16.2.10`.
- Supabase JS: `2.105.1` ← `2.110.2`.
- اختفت ثغرتا التدقيق العاليتان وسلسلة WebSocket المتأثرة.
- بقي تنبيهان متوسطان في PostCSS المضمّن داخل Next.js، ولا يقدم npm إصلاحًا
  غير كاسر حاليًا؛ تمنع CI أي تنبيه جديد بدرجة عالية أو حرجة.
- لا تستخدم `npm audit fix --force`؛ اقتراحه الحالي يخفض Next.js إلى إصدار
  قديم كاسر وغير مقبول.
