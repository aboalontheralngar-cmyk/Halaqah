# تسليم المرحلة P2.2 — اعتماد نماذج الاختبارات

التاريخ: 2026-07-11

## ما نُفذ

- حفظ نموذج الاختبار وأسئلته في SQLite مع ترقية قاعدة البيانات إلى الإصدار 6.
- إدخال القوالب والأسئلة ضمن النسخ الاحتياطي والاستعادة، مع توافق النسخ القديمة.
- تعديل نطاق السؤال ونوعه، واستبداله، وحذفه، وتحريكه قبل اعتماد النموذج.
- سجل قوالب قابل لإعادة الفتح والتحديث والحذف والطباعة بمقاسي A4 وA5.
- مزامنة القوالب والأسئلة والحذف المؤجل إلى Supabase.
- migration تراكمي يضيف نطاق الطالب والحلقة وسياسات RLS الآمنة.
- توسيع فحص عقد الأمان ليشمل القوالب ومسار مزامنتها.

## نتائج التحقق

- `npm run validate:quran`: ناجح.
- `npm run validate:security`: ناجح.
- `npx tsc --noEmit`: ناجح.
- `npm run build`: ناجح، وجميع مسارات الويب الـ25 بُنيت.
- `git diff --check`: ناجح.

## ما يحتاج بيئة خارجية

- تشغيل `flutter analyze` و`flutter test` وبناء APK تجريبي؛ Flutter SDK غير متوفر في بيئة العمل الحالية.
- اختبار طباعة A4 وA5 على طابعة فعلية.
- تطبيق migrations على قاعدة Supabase مرحلية أولًا، ثم اختبار مدير ومعلمين في حلقتين مختلفتين قبل الإنتاج.
