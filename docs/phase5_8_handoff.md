# تسليم المرحلة P5.8 — نظام التصميم وتجربة الاستخدام

التاريخ: 2026-07-13

## النتيجة

أصبح للمشروع نظام تصميم مشترك بدل أن تحدد كل شاشة ألوانها ومسافاتها
وبطاقاتها منفردة. يشمل ذلك الوضع الفاتح والداكن، والمكونات اليومية، والتنقل
على الهاتف، مع تطبيق فعلي على أكثر الشاشات استعمالًا في Android والويب.

## Android

- `AppSpacing` و`AppRadii` و`AppBreakpoints` مرجع موحد للمسافات والحواف
  والاستجابة.
- الثيم ينسق البطاقات والحقول والأزرار والحوارات والشرائح والتنبيهات
  وأشرطة التنقل في الوضعين.
- مكونات مشتركة لرأس الصفحة وعنوان القسم والحالة الفارغة والبحث والمؤشر.
- شاشة الطلاب تستخدم البحث والحالة الفارغة وبطاقة الطالب المشتركة.
- شاشة التقارير تستخدم رأسًا واضحًا وعناوين أقسام وبطاقات طلاب موحدة.
- شاشة الحضور لا تفرض خلفية بيضاء في الوضع الداكن وتستخدم الحالة الفارغة
  المشتركة.

## الويب

- `AppDesign.tsx` يوفر `PageHeader` و`Surface` و`MetricCard` و`EmptyState`
  و`SearchField`.
- صفحات الطلاب والحضور والتسميع تستخدم المكونات المشتركة.
- شريط الهاتف السفلي لا يستهلك مساحته بأولياء الأمور والانضباط؛ يعرض
  الوظائف اليومية الخمس، مع بقاء القائمة الجانبية كاملة.
- أضيفت حالة `aria-current` وتسميات أزرار القائمة والوضع الداكن.
- أضيف تركيز لوحة المفاتيح واحترام `prefers-reduced-motion`.

## SQL

لا يوجد SQL جديد في هذه المرحلة. لا تنفذ أي migration بسبب P5.8.

## التحقق

- `npm run validate:design`
- جميع فحوص العقود الموجودة في `website/scripts`.
- `git diff --check`.
- `npm run build`: نجح TypeScript وتوليد 26 مسارًا.
- فحص ESLint للملفات الأساسية المعدلة بلا أخطاء (أربعة تحذيرات hooks قديمة)،
  بينما الفحص الشامل ما زال يكشف 51 خطأ متراكمًا في ملفات أخرى؛ أدرج علاجها
  ضمن مرحلة P6 قبل جعل lint بوابة نشر إلزامية.
- Flutter SDK غير متاح في بيئة العمل؛ يلزم `flutter analyze` و`flutter test`
  ومراجعة الوضعين على جهاز قبل الإصدار.

## اختبار القبول المقترح

1. بدّل بين فاتح وداكن وأعد فتح التطبيق وتأكد من حفظ الاختيار.
2. افتح الطلاب والحضور والتقارير على هاتف صغير وجهاز لوحي.
3. اختبر البحث دون نتائج وتأكد من وضوح الحالة الفارغة.
4. في الويب، تنقل من الشريط السفلي إلى الحفظ والتقارير ثم افتح القائمة
   الجانبية لبقية الأقسام.
5. اختبر التنقل بلوحة المفاتيح وقارئ الشاشة بصورة سريعة.
