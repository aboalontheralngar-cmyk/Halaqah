# تسليم المرحلة P6.4 — التشخيص الآمن والجاهزية التشغيلية

## النتيجة

- أصبح للتطبيق مركز تشخيص من الإعدادات يعرض إصدار التطبيق وSQLite والنظام.
- يعرض أعداد الجداول التشغيلية دون أسماء أو معرفات أو تفاصيل طلاب.
- يجمع حالة النسخ المحلية وآخر رفع وتنزيل واتجاه المزامنة وفحص Supabase.
- يمكن نسخ تقرير الدعم أو مشاركته كنص منقح.
- تلتقط معالجات Flutter والمنصة الأخطاء غير المعالجة كبصمة SHA-256 مختصرة.
- لا يخزن الحدث نص الاستثناء أو stack trace أو مسار الجهاز.
- تمنع مهلة خمس دقائق تكرار الحادثة نفسها وإغراق سجل التدقيق.
- عند فشل التهيئة تظهر شاشة تعافٍ برمز حادثة بدل شاشة فارغة، وتحذر من حذف البيانات.
- الإصدار الحالي `4.1.0-alpha.2+42` باسم `RC2`.

## اختبار الجهاز

1. افتح الإعدادات > التشخيص > مركز التشخيص والدعم.
2. تأكد أن إصدار SQLite هو 18 وأن اتصال Supabase سليم.
3. انسخ التقرير وابحث فيه عن اسم طالب أو هاتف؛ يجب ألا يظهر أي منهما.
4. تأكد أن أعداد الطلاب والحضور منطقية دون إظهار تفاصيل السجلات.
5. نفّذ `flutter test` للتحقق من تنقيح البصمة ومنع التفاصيل الخام.

## الخصوصية

تقرير الدعم يعرض معلومات تشغيلية فقط. لا ترسل معه لقطة شاشة من بيانات طالب،
ولا تضف أسماء أو هواتف يدويًا عند فتح تذكرة دعم.

## SQL

لا يوجد SQL جديد. تستعمل الشاشة SQLite المحلي وفحص HTTPS الحالي فقط.
