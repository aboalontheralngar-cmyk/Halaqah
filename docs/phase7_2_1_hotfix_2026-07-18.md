# إصلاح P7.2.1 — Supabase وتوافق ملفات Flutter

التاريخ: 2026-07-18

## المشكلة الأولى: `gen_random_bytes(integer) does not exist`

قد يثبت Supabase إضافة `pgcrypto` داخل مخطط `extensions` بدل `public`. كانت
دوال البوابة تستخدم `search_path = public, pg_temp`، ولذلك لم تستطع دالة SQL
العثور على `gen_random_bytes` رغم وجود الإضافة.

أصبح المسار الآن:

```sql
SET search_path = public, extensions, pg_temp
```

ويسري الإصلاح على دوال بوابة العائلة، كما يصلح إعداد دوال بوابة الطالب المثبتة
سابقًا عند تنفيذ migration P7.2.1 المصحح.

## المشكلة الثانية: أخطاء Flutter المتتابعة

كانت `database_service.dart` و`supabase_service.dart` من إصدار حديث، بينما كانت
النسخة المحلية تفتقد أو تستخدم نسخًا أقدم من الملفات التالية:

- `lib/services/recitation_points_policy.dart`
- `lib/models/student.dart`
- `lib/models/fund_transaction.dart`

لهذا ظهر فقد `RecitationPointsResult`، ثم `studentCode` و`reviewPlanAmount`، ثم
`behaviorPointId`. هذه ليست أخطاء مستقلة؛ كلها نتيجة تحديث جزئي غير مكتمل.

## طريقة التطبيق

1. أغلق التطبيق وVisual Studio Code إن كان يقفل الملفات.
2. فك حزمة Hotfix فوق جذر مشروع `Halaqah` ووافق على استبدال الملفات.
3. في Supabase SQL Editor افتح استعلامًا جديدًا والصق **محتوى** ملف
   `20260714000600_p7_family_portal.sql` المصحح كاملًا، ثم شغله مرة واحدة.
4. من جذر المشروع، وليس مجلد `website`، نفذ:

```powershell
flutter clean
flutter pub get
flutter analyze
flutter test
flutter run
```

لا يلزم تنفيذ `flutter upgrade` لحل هذه الأخطاء.

## ملاحظة سلامة

يبدأ migration بـ`BEGIN` وينتهي بـ`COMMIT`. الخطأ السابق ألغى المعاملة، والملف
المصحح قابل لإعادة التنفيذ؛ لا تحذف الجداول ولا تنفذ مخطط التثبيت الجديد.
