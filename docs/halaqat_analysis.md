# 🔍 تحليل مقارن: مشروع Halaqat (AdamKourchi) vs مشروعنا Halaqah

## نظرة عامة على مشروع Halaqat

مشروع **Halaqat** هو تطبيق متكامل لإدارة حلقات تحفيظ القرآن الكريم، مبني بتقنيات:
- **Angular 20** + **Ionic 8** + **Capacitor 8**
- قاعدة بيانات **SQLite** محلية بالكامل (بدون إنترنت)
- تصدير التقارير بـ **ExcelJS**
- مشاركة/استيراد الحلقات عبر **JSON**

## 🌟 الميزات المتقدمة في Halaqat التي تستحق الاستلهام

### 1. 📝 نظام الواجبات والتسميع المتقدم (الأولوية: عالية جداً 🔴)

هذه أهم ميزة في Halaqat. النظام يتضمن:

- **تحديد نطاق الآيات**: اختيار السورة والآية (بداية ونهاية) لكل واجب
- **نظام التقييم**: مقياس 5 مستويات:
  - `Excellent` (ممتاز) = 4 نقاط
  - `Very Good` (جيد جداً) = 3 نقاط
  - `Good` (جيد) = نقطتان
  - `Needs Work` (يحتاج مراجعة) = نقطة واحدة
  - `Absent` (غائب) = 0
- **عدد الأخطاء** لكل واجب
- **ملاحظات** حرة
- **ربط بالأحزاب/الأثمان** تلقائياً

> **الفائدة لمشروعنا**: إضافة شاشة "تسميع" كاملة حيث يحدد المعلم نطاق الآيات ويقيّم أداء الطالب.

---

### 2. 🗺️ خريطة المصحف المرئية - Mushaf Visualizer (الأولوية: عالية 🟠)

ميزة فريدة: تعرض خريطة بصرية لتقدم الطالب عبر **60 حزب × 8 أثمان = 480 خلية**:

- **لون أخضر**: محفوظ بشكل جيد
- **لون أصفر**: بحاجة مراجعة (Aging - 14-30 يوم)
- **لون أحمر**: متراجع (Stale - أكثر من 30 يوم)
- **لون رمادي**: لم يُحفظ بعد

```
نظام تتبع التراجع (Decay):
- FRESH:  آخر تقييم < 14 يوم
- AGING:  آخر تقييم 14-30 يوم
- STALE:  آخر تقييم > 30 يوم
```

> **الفائدة لمشروعنا**: واجهة رائعة تُظهر للمعلم وولي الأمر نظرة شاملة على مستوى الطالب في حفظ القرآن.

---

### 3. 📊 تقارير Excel المتقدمة (الأولوية: عالية 🟠)

نظام تقارير متطور باستخدام **ExcelJS**:

- **تقرير فردي**: سجل أداء طالب واحد
- **تقرير حلقة**: جميع طلاب حلقة واحدة
- **تقرير موحد**: عدة حلقات دفعة واحدة
- **دعم RTL**: الكتابة من اليمين لليسار
- **تجميع أسبوعي**: يجمع التواريخ في أسابيع (الأسبوع الأول، الثاني...)
- **حساب مجموع الحفظ**: عدد الأثمان/الأحزاب لكل أسبوع
- **مشاركة فورية**: عبر واتساب وتطبيقات أخرى

> **الفائدة لمشروعنا**: إضافة زر "تصدير التقرير" في ملف الطالب وصفحة الحلقة.

---

### 4. 🔗 مشاركة واستيراد الحلقات (الأولوية: متوسطة 🟡)

- **تصدير**: تحويل حلقة كاملة (طلاب + واجبات) إلى ملف JSON
- **استيراد**: استقبال ملفات JSON من أجهزة أخرى
- **Intent Filters**: فتح تلقائي لملفات JSON
- **حلقات مشتركة**: تبويب خاص للحلقات المستوردة (للقراءة فقط)

> **الفائدة لمشروعنا**: تبادل البيانات بين المعلمين أو نقلها بين أجهزة.

---

### 5. 💬 قوالب الرسائل (الأولوية: متوسطة 🟡)

- **قالب الواجب**: نص مُعد مسبقاً لإرسال الواجب لولي الأمر عبر واتساب
- **قالب التقييم**: نص مُعد مسبقاً لإرسال نتيجة التقييم
- **متغيرات ديناميكية**: يتم استبدال اسم الطالب والسورة والتقييم تلقائياً

> **الفائدة لمشروعنا**: تسهيل التواصل مع أولياء الأمور بنقرة واحدة.

---

### 6. 📏 تقسيمات القرآن المتقدمة (الأولوية: عالية 🟠)

نظام `quran_divisions` المتقدم:

- **جدول `quran_divisions`**: 60 حزب × 8 أثمان = 480 صف
- **تحويل ثنائي الاتجاه**:
  - حزب + ثُمن → نطاق آيات (سورة:آية)
  - نطاق آيات → حزب + ثُمن
- **حساب عدد الأثمان** بين نطاقين من الآيات

> **الفائدة لمشروعنا**: أساس لبناء خريطة المصحف ونظام الواجبات المتقدم.

## 🎯 خطة التطوير المقترحة (مرتبة بالأولوية)

### المرحلة 1: الواجبات والتسميع 🔴
1. إضافة نموذج `Homework` للـ database
2. بناء شاشة "تسميع" مع محدد السورة/الآية
3. نظام التقييم (5 مستويات)
4. ربط الواجبات بالطالب وعرضها في ملفه

### المرحلة 2: تقسيمات القرآن + خريطة المصحف 🟠
5. إضافة جدول `quran_divisions` (480 صف)
6. بناء `MushafProgressService` 
7. بناء واجهة خريطة المصحف المرئية
8. نظام تتبع التراجع (Decay)

### المرحلة 3: تقارير Excel 🟠
9. إضافة مكتبة Excel خفيفة (csv بدلاً من ExcelJS للحفاظ على الخفة)
10. تقرير فردي للطالب
11. تقرير حلقة كاملة
12. مشاركة التقارير

### المرحلة 4: قوالب الرسائل + مشاركة واتساب 🟡
13. قالب إرسال الواجب لولي الأمر
14. قالب إرسال التقييم
15. زر مشاركة واتساب مباشر

### المرحلة 5: تصدير/استيراد الحلقات 🟡
16. تصدير الحلقة كـ JSON
17. استيراد حلقة من ملف JSON
18. تبويب "حلقات مشتركة"
