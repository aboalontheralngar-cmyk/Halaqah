# تسليم المرحلة P7.2 — بوابة الطالب وولي الأمر

التاريخ: 2026-07-14

## النتيجة

أضيفت بوابة عربية RTL للقراءة فقط على `/portal`. يدخل الطالب أو ولي الأمر
بكود الطالب وPIN من ستة أرقام، ثم يرى الخطة والحضور والحفظ والمراجعة والنقاط
لفترة يحددها، ويستطيع طباعة الصفحة أو حفظها PDF.

## ضمانات الأمان

- كود الطالب وQR ليسا كلمة مرور.
- PIN مخزن بتجزئة bcrypt وملح عشوائي، ولا يعود إلى التطبيق بعد حفظه.
- لا يصل المتصفح إلى جداول الطالب مباشرة ولا يحمل `service_role`.
- Edge Function تتحقق من المدخلات وبصمة العميل وحد المحاولات.
- الجلسة 12 ساعة، والقاعدة تخزن SHA-256 للرمز فقط.
- تغيير PIN أو إيقاف البوابة يلغي الجلسات القديمة فورًا.

## خطوات النشر

1. خذ نسخة احتياطية.
2. نفّذ محتوى `20260714000400_p7_student_portal_security.sql` كاملًا.
3. من مجلد `website` اربط Supabase CLI بالمشروع.
4. عرّف `PORTAL_RATE_LIMIT_PEPPER` بقيمة عشوائية طويلة لا تقل عن 32 بايت.
5. عرّف `PORTAL_ALLOWED_ORIGINS` بعنوان الموقع، مثل `https://halaqah.example.com`.
6. انشر Function باسم `student-portal` ثم انشر موقع Next.js.
7. فعّل PIN لطالب تجريبي من ملفه، واختبر الدخول والخروج والقفل والطباعة.

أمثلة أوامر CLI بعد استبدال القيم:

```bash
supabase secrets set PORTAL_RATE_LIMIT_PEPPER="RANDOM_LONG_SECRET"
supabase secrets set PORTAL_ALLOWED_ORIGINS="https://YOUR_WEB_DOMAIN"
supabase functions deploy student-portal --no-verify-jwt
```

## قبل فتحها للعامة

- اختبر طالبين من مركزين مختلفين وتأكد أن كل جلسة ترى طالبًا واحدًا فقط.
- جرّب خمسة أرقام خاطئة وتأكد من ظهور القفل المؤقت.
- غيّر PIN أثناء وجود جلسة مفتوحة وتأكد أن الجلسة القديمة تتوقف.
- لا تستخدم بيانات الإنتاج لأول اختبار، ولا ترسل PIN داخل سجل أو مجموعة عامة.

## المؤجل

حساب عائلي واحد لعدة أبناء، واستعادة PIN ذاتية، وCAPTCHA، وتقارير منشورة ثابتة
هي نطاق P7.2.1 بعد اختبار النسخة الأولى.
